## Base URL

```
https://api.ocrbase.dev/v1
```

For self-hosted instances, replace with your server URL.

## Authentication

All API requests require a Bearer token. Create an API key from the dashboard or via the [Keys API](/docs/api/keys).

```bash
curl https://api.ocrbase.dev/v1/parse \
  -H "Authorization: Bearer sk_your_api_key" \
  -F file=@document.pdf
```

## Async processing

Parse and extract endpoints return immediately with a `202 Accepted` response containing the job object. Poll `GET /v1/jobs/{id}` or subscribe via [WebSocket](/docs/api/realtime) for updates.

```
POST /v1/parse  →  202 { id: "job_xxx", status: "pending", ... }

GET /v1/jobs/job_xxx  →  200 { status: "processing", ... }
GET /v1/jobs/job_xxx  →  200 { status: "completed", markdownResult: "...", ... }
```

## Error responses

Errors return a JSON body with `message`:

```json
{
  "message": "Unauthorized"
}
```

| Status | Description                               |
| ------ | ----------------------------------------- |
| `400`  | Bad request — invalid input               |
| `401`  | Unauthorized — missing or invalid API key |
| `404`  | Not found                                 |
| `429`  | Rate limited                              |
| `500`  | Server error                              |
| `503`  | Service unavailable — OCR service down    |
